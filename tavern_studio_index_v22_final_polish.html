MVU 编辑器

第二版新增：专门管理状态变量、初始状态、更新规则和可见状态栏。
新手说明：MVU 和状态栏到底是什么
普通状态栏 只是让 AI 在回复末尾显示“时间、地点、情绪、好感”等信息。优点是简单，缺点是容易忘记或格式漂移。
MVU 变量系统 可以理解为“角色当前状态的数据表”。每个变量有 key、初始值、更新规则，AI 按剧情变化更新。
隐藏状态块 给 AI 看的状态数据，通常用 JSON。用户看到的是美化后的状态栏，AI 看到的是更稳定的数据结构。
变量 key：建议用英文，例如 affection、location、desire。不要用空格，不要频繁改名。
显示名称：给用户看的中文名，例如“好感度”“当前位置”“占有欲”。
类型：number 是数字，text 是文字，boolean 是是/否，enum 是固定选项，object 适合复杂状态。
更新规则：写清楚什么时候增加、减少、改变。例如“亲密互动后小幅上升，争吵后下降”。

初始状态 JSON

把所有变量开局状态放在一起。新手可以先留空，或写简单 JSON。

可见状态栏模板

这是玩家看到的状态栏样式。可以写成卡片、列表、诗意文字或简洁表格。

隐藏状态块格式

安装说明 / 备注

写给自己或朋友看的使用方法，例如应该把状态规则放进哪里、怎么复制状态栏。

角色卡 JSON 导出 / 校验器

把 AI 生成内容、旧角色卡或手写 JSON 检查并整理为 SillyTavern V2 结构。
新手说明：这个校验器做什么
不是内容审美评分主要检查 JSON 结构、关键字段、数组/对象格式，避免导入酒馆时因为格式出错。
可以包装为 V2如果你粘贴的是普通对象，校验器会尝试整理成 spec: chara_card_v2 的结构。
不会强制填满字段缺失字段会提示警告，但不会阻止导出。角色卡可以之后继续完善。

结构预览

导出包管理

统一整理角色卡、世界书、MVU 和安装说明。注意：编辑器内容用于手动整理；AI 生成导出优先使用当前生成流程产出的工程结构。
新手说明：导出包怎么用
角色卡 JSON 优先来自“导出校验器”里的 V2 结构。建议先去校验器包装并检查一次。
世界书 JSON 来自“世界书编辑器”。如果角色卡内嵌世界书不够完整，可以单独导入这个文件。
MVU / 状态栏 来自“MVU 编辑器”或状态栏生成结果。不同酒馆玩法需要按安装说明复制到对应位置。

模板填写

你手动填，AI 可补全、优化、转角色卡 JSON。
新手说明：这个模板怎么填
不用每项都填满 没想法的字段可以留空，不做硬性要求。之后点“让 AI 补全”。你只需要先填最重要的身份、关系、性格、世界观。
字段不是越长越好 酒馆角色卡更重视稳定人设。每项写清楚“是什么、为什么、怎么表现”比堆形容词更有用。
先写骨架，再让 AI 润色 例如先写“旧情人、现代都市、表面温柔实际控制欲强”，再让 AI 扩写成完整设定。
推荐先填 核心概念、关系、性格、背景故事、说话风格、开场气氛。
可以后填 生日、身高、服装细节、家庭成员、具体喜好。
NSFW 字段 只用于虚构成年人角色。可以留空，也可以写边界、偏好、限制和亲密互动规则。
这里可以写你希望 AI 怎么处理整张角色卡。比如“重点扩写背景故事”“把性格改得更病态克制”“只润色说话风格，不改基础设定”。不填也可以。
